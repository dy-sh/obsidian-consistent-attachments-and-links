# Settings

Open **Settings -> Community plugins -> Consistent Attachments and Links** to configure the plugin. Each option below lists the setting key stored in the plugin's `data.json`.

> [!WARNING]
>
> Several of these settings change files on disk. The plugin shows a backup warning and, until you acknowledge it, keeps the most destructive options disabled. Back up a real vault before turning them on.

## What happens when notes move, rename, or delete

- `shouldMoveAttachmentsWithNote` - move a note's attachments along with the note when it is moved.
- `shouldDeleteAttachmentsWithNote` - delete attachments that become unused when their note is deleted.
- `shouldUpdateLinks` - rewrite links so they keep resolving after a rename or move.
- `shouldDeleteExistingFilesWhenMovingNote` - when moving into a folder that already has an attachment of the same name, delete the moved duplicate instead of renaming it.
- `shouldChangeNoteBacklinksDisplayText` - keep the display text of backlinks in sync when a note is renamed.
- `shouldCollectAttachmentsAutomatically` - collect a note's attachments into its folder automatically as you edit, without running a command.

## Attachment collecting and moving

- `collectAttachmentUsedByMultipleNotesMode` - what to do when a collected attachment is used by several notes: `Skip`, `Copy`, `Move`, `Cancel`, or `Prompt`.
- `moveAttachmentToProperFolderUsedByMultipleNotesMode` - the same choice for the **Move Attachment to Proper Folder** command: `Skip`, `CopyAll`, `Cancel`, or `Prompt`.
- `excludePathsFromAttachmentCollecting` - vault paths that attachment-collecting commands must leave untouched.

## Scope: which paths the plugin acts on

- `includePaths` - restrict the plugin's consistency handling to these paths (empty means the whole vault).
- `excludePaths` - paths the plugin ignores entirely.

## Reports, folders, and safety

- `consistencyReportFile` - path of the note generated by **Check Vault Consistency** (default `consistency-report.md`).
- `emptyFolderBehavior` - how folders left empty after a move are handled (for example delete them, delete them together with empty parents, or keep them).
- `treatAsAttachmentExtensions` - extensions that should be treated as attachments even though they are Markdown, such as `.excalidraw.md`.
- `shouldShowBackupWarning` - show the backup warning for destructive operations; while it is on, the plugin reverts the most dangerous settings on load as a safety net.

Change any of these, then work through [[01 Attachments move with their note]], [[02 Links stay valid on rename and move]], [[03 Check vault consistency]], and [[04 Reorganize and convert links]] to watch them take effect.
